# Play Store Assets

This directory contains all the assets required for publishing TalkAR on the Google Play Store.

## Required Assets

### Screenshots (3)

- Main screenshot showing AR avatar overlay
- Screenshot of the camera interface
- Screenshot of the avatar speaking

### App Icon

- High-resolution transparent PNG (512x512)

### Feature Graphic

- 1024x500 promotional image

### Descriptions

- Short app description (80 characters max)
- Full description (4000 characters max)

### Privacy Policy

- Link to privacy policy document

### Content Rating

- Answers to content rating questions

## Asset Specifications

### Screenshots

- Format: JPG or PNG
- Minimum dimension: 320px
- Maximum dimension: 3840px
- Aspect ratio: Up to 16:9
- File size: Up to 8MB each

### App Icon

- Format: PNG
- Dimensions: 512x512 pixels
- No transparency (will be placed on colored backgrounds)
- No rounded corners (these will be applied automatically)

### Feature Graphic

- Format: JPG or PNG
- Dimensions: 1024x500 pixels
- Safe zone: Keep important content within 910x500 pixels
- File size: Up to 1MB

## Asset Files

1. `screenshot_1.png` - Main AR experience
2. `screenshot_2.png` - Camera interface
3. `screenshot_3.png` - Avatar speaking
4. `app_icon.png` - 512x512 app icon
5. `feature_graphic.png` - 1024x500 promotional image
6. `short_description.txt` - Brief app description
7. `full_description.txt` - Detailed app description
8. `privacy_policy_url.txt` - Privacy policy link
9. `content_rating_answers.txt` - Content rating questionnaire answers

## Guidelines

### Short Description

- Max 80 characters
- Clear and concise
- Highlight key features
- Include relevant keywords

### Full Description

- Max 4000 characters
- Start with most important information
- Use clear, simple language
- Include features and benefits
- Mention supported devices
- Include contact information

### Screenshots Tips

- Show the app in action
- Highlight unique features
- Use high-quality images
- Include text overlays if needed
- Maintain consistent visual style

## Updates

This directory should be updated whenever:

- App UI changes significantly
- New features are added
- Descriptions need updating
- Privacy policy changes
